# Spec 003 — Go event-trigger SDK: codegen + handler contract

Status: design approved (2026-06-13), not yet implemented.

Let users handle Donat event triggers in a Go project — not only via webhooks
but by registering a native Go function per trigger. This spec covers the
**transport-agnostic core** only: (1) generated per-table Go structs, and
(2) the `Event[T]` envelope + handler registry that any transport dispatches
into. The delivery transport itself (sidecar pull vs in-process wasm) is
**explicitly deferred** — see [[knowledgebase/embedded-sdk/_index]] and
ADR-001/005. Both pieces here are identical regardless of how events arrive,
which is exactly why they can be built first.

Builds directly on Spec 002 (table event triggers): the journal, the Donat
event envelope, and the YAML `event_triggers` declaration already exist. This
spec adds a Go consumer for that same envelope.

## Scope

In scope:

1. **`donat codegen go`** — Rust subcommand that introspects the catalog,
   filters to tracked tables, and emits Go row structs.
2. **Go SDK core** (`sdk/go/donat`): the `Event[T]` envelope type (Donat
   shape, `op`/`old`/`new`/`session`), a name→handler registry, and a single
   transport-agnostic `Dispatch` seam.

Out of scope (deferred, separate specs):

- Any transport: webhook receiver adapter, sidecar pull loop, in-process
  wasm-core. They will all call `Dispatch`; none is built here.
- Sync `pre_insert` / `post_insert` hooks. The registry is designed so they
  slot in later (see Forward-compatibility), but they are not implemented now.
- Custom functions / Actions.

## 1. Codegen: `donat codegen go`

New subcommand on the `donat` binary (`crates/server`), alongside `migrate` /
`validate`. Deploy-time, never at serving time.

```
donat codegen go --metadata-dir <dir> --out <dir> [--package donat_gen]
```

- **Source of truth: pg catalog.** Reuse `crates/catalog` introspection for
  column names, pg type, nullability. The event payload (`to_jsonb(OLD/NEW)`)
  carries raw DB column values keyed by DB column name, before permissions —
  so the generated struct mirrors the physical table, not a per-role schema.
- **Table set:** only tables tracked in the metadata directory
  (`--metadata-dir`), so generated code matches what the engine serves.
- **One struct per table**, exported, `PascalCase` of the table name. One
  field per column, exported, `PascalCase`, with `json:"<db_column_name>"`
  tags (the names `old`/`new` use over the wire).

### pg → Go type mapping

| pg type | Go type |
|---|---|
| `int2` | `int16` |
| `int4` | `int32` |
| `int8` | `int64` |
| `numeric` / `decimal` | `decimal.Decimal` (`github.com/shopspring/decimal`) |
| `float4` / `float8` | `float64` |
| `text` / `varchar` / `bpchar` / `name` | `string` |
| `bool` | `bool` |
| `timestamptz` / `timestamp` | `time.Time` |
| `date` | `time.Time` |
| `uuid` | `string` |
| `jsonb` / `json` | `json.RawMessage` |
| array `_T` | `[]<Go T>` |
| pg enum | typed `string` alias + `const` per label |
| unknown / unmapped | `json.RawMessage` (never fail codegen; log a warning) |

- **Nullability:** a column nullable in the catalog → pointer to the mapped
  type (`*string`, `*decimal.Decimal`, …). `NOT NULL` → value type.
  Rationale: in `old`/`new`, individual nullable columns can be JSON `null`;
  whole-row absence (`old` on INSERT, `new` on DELETE) is handled at the
  envelope level (`*T`), not per field.
- **`numeric → decimal.Decimal` is the API-consistent choice**, confirmed
  against the engine: `to_jsonb` emits numeric as a full-precision JSON
  number, and `decimal.Decimal.UnmarshalJSON` accepts both JSON numbers and
  strings — so it also matches the GraphQL path under
  `stringify_numeric_types` either way, with no precision loss. `float64`
  would lose precision exactly in the mode Donat's stringify flag exists to
  protect.
- **Enums:** introspect `pg_enum`; emit `type <Name> string` and a `const`
  block of labels. Column of that enum type → the alias (pointer if nullable).
- Output: a single `donat_gen.go` in `--out` (one package), gofmt-clean,
  with a `// Code generated by donat codegen go; DO NOT EDIT.` header.
  Multi-schema name collisions are disambiguated by prefixing the schema
  (e.g. `AuthUser` for `auth.user`); `public` is never prefixed.

## 2. Go SDK core (`sdk/go/donat`)

Pure-Go package, no cgo, no transport. Module path TBD at scaffold time
(placeholder `github.com/donat/donat-go`).

### Event envelope

Hand-written generic mirroring the Donat envelope from Spec 002 (§4). `T` is a
generated row struct.

```go
type Op string
const (OpInsert Op = "INSERT"; OpUpdate Op = "UPDATE"; OpDelete Op = "DELETE")

type Event[T any] struct {
    ID        string
    CreatedAt time.Time
    Table     TableRef          // {Schema, Name}
    Trigger   TriggerRef        // {Name}
    Op        Op                // from event.op
    Old       *T                // event.data.old — nil on INSERT
    New       *T                // event.data.new — nil on DELETE
    Session   map[string]string // event.session_variables (may be nil; see Spec 002 limitation)
    Delivery  DeliveryInfo      // {CurrentRetry, MaxRetries}
}
```

`Event[T]` has a custom `UnmarshalJSON` (or a `ParseEvent[T](raw []byte)`
helper) that maps the nested Donat envelope (`event.op`,
`event.data.old/new`, `event.session_variables`, top-level `id`/`created_at`/
`table`/`trigger`/`delivery_info`) onto the flat struct. This is the only
place that knows the wire shape.

### Registry + Dispatch seam

```go
type Registry struct { /* name -> typed dispatcher */ }

// Generic registration helper (typed; the only API users touch):
func On[T any](r *Registry, triggerName string, h func(context.Context, Event[T]) error)

// Transport-agnostic entry point. Every future transport (webhook receiver,
// pull loop, in-process) decodes nothing itself — it hands the raw envelope
// body here:
func (r *Registry) Dispatch(ctx context.Context, triggerName string, rawEnvelope []byte) error
```

- `On[T]` stores a closure that does `ParseEvent[T]` then calls `h`. The
  registry holds `func(ctx, []byte) error` internally, so `Dispatch` is
  non-generic and any transport can call it.
- `Dispatch` for an unknown `triggerName` returns a typed error
  (`ErrNoHandler`) — the transport decides whether that is fatal.
- **Startup validation (transport's job, but provided here):**
  `Registry.Names()` exposes registered names so a transport/boot check can
  assert every YAML `event_triggers[].name` has a handler and fail fast.
- Concurrency contract documented now (handlers may be called concurrently;
  must be concurrent-safe) — carries over to every transport.

No `net/http`, no pull loop, no `Start`/`Run` in this spec. Those belong to a
transport spec and call `Dispatch`.

## Forward-compatibility (pre/post sync hooks, later)

The registry is the seam that keeps sync hooks from being a parallel system:

- Same `Registry`, same name-keyed registration. Sync hooks add
  `OnPreInsert[T](r, schema, table, func(ctx, rows []T) ([]T, error))` etc.
- Difference is the **call site and signature**, not the registration model:
  async `On` returns `error` (retry); sync returns modified rows / rejection
  and is invoked in-transaction by the engine before commit.
- `Event[T]` (`old`/`new`) is the async contract; sync hooks get row slices.
  Keeping both under one registry means a future transport wires both at once.

## Testing

- **Codegen (Rust, `crates/server` or a `codegen` module):** unit/insta
  snapshot tests over a fixture catalog — assert generated Go for: each
  scalar, nullable→pointer, array, enum (alias+consts), `numeric→decimal`,
  unknown→`json.RawMessage`. Snapshots reviewed, never blind-accepted.
- **Go SDK (`go test ./sdk/go/...`):** table-driven `ParseEvent[T]` tests
  using real envelope JSON from Spec 002 (INSERT→`Old==nil`,
  DELETE→`New==nil`, UPDATE→both; numeric precision round-trip via
  `decimal.Decimal`; null column → nil pointer). `Dispatch` routing +
  `ErrNoHandler`. No transport, so no network in tests.
- The generated structs must compile against the SDK (`go build ./...` on a
  golden generated package in testdata).

## Open items (resolved → for the record)

- numeric → `decimal.Decimal` (not string, not float64) — chosen for
  precision and API-consistency.
- Transport — deferred entirely; `Dispatch` is the agreed seam.
- Session variables may be nil until Spec 002's GUC limitation is addressed.
